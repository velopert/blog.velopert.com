이전에 사용하던 블로그는 워드프레스 블로그인데, 뭐 나름대로 잘 사용하고 있긴 했으나, 게시글을 요즘 마크다운으로 관리를 하고 싶어져서 이렇게 새 블로그를 준비하고 있다. 기존의 워드프레스 블로그는 주요 포스트를 업데이트 하면서 옮겨오고, 방문자 수가 비슷비슷해질 때 까지 운영 할 예정이다. 

기존에 워드프레스에 쓴 글이 좀 많긴 한데.. 업데이트 하는게 무리 일 것 같으면 그냥 복사 붙여넣기라도 해야겠다. 업데이트는 추후 하면 되니까. 블로그 마이그레이션이 끝날 때 까지는, 새 포스트를 작성 할 때 마다, 워드프레스에도 올리고 여기에도 올릴 계획이다.

물론 워드프레스에서도 마크다운으로 포스트를 작성 할 수 있긴 하다. 하지만, 마크다운을 사용 할 때, 이용 할 수 있는 SyntaxHighlighter 플러그인이 좀 제한된다. 좀 맘에 안드는 부분도 있기도 하고... 추가적으로, 1년정도 블로그를 운영하고나니, 블로그에 방문자가 좀 늘어나서, 기존의 호스팅으론 가끔씩 데이터베이스 서버가 가끔씩 맛이 가는 경우가 있었다. 블로그 규모가 커질수록 계속 서버를 업그레이드하는건 뭔가 아깝기도하고 무리가 될 수도 있으니, 정적으로 파일들을 제공하는 방식으로 하는것도 괜찮을것같다고 생각한다.

솔직히 조금 귀찮긴 하지만, 나중을 생각한다면, 결국엔 잘 한 선택일 것 이라고 생각한다.

블로그 플랫폼을 Hexo 로 할 지, Jekyll 로 할 지 고민을 좀 했는데, 인기도만 따지자면, Jekyll 이 훨씬 높아보이지만, 아무래도, Ruby 는 1 도 모르다 보니까, NodeJS 기반인 Hexo 로 하는게 나을 거라는 생각을 했다. 혹시 내가 나중에 커스터마이징을 한다면, 그래도 익숙한 환경에서 작업하는게 나으니까. 일단 당분간은 테스트삼아 여기서 포스팅을 몇번 해보고, 맘에 안드면 그냥 워드프레스로 돌아가던지.. 해야겠다.

만약에 완전히 Hexo 로 옮겼을 때, 한가지 아쉬운점이 있다면, 워드프레스 블로그 포스트에 남아있던 덧글들? 물론 Disqus 를 사용 하였기에, 데이터가 어디로 날아가지는 않겠지만.. 그래도, 더 좋은 블로그를 위하여 그 정도는 감수해야 하려나..?

특히, 2016을 마치며 썼던 2016.log 포스트에서는 많은 블로그 구독자분들께서 응원의 글을 남겨주셨는데 이거는 살리고 싶다. 다행히도, disqus 에서 좀 알아보니 migration tool 이 있긴 있다. 

여하튼, 이 블로그는 blog.velopert.com 이라는 도메인으로 운영을 해야겠다. 나중에가서 velopert.com 으로 다시 설정하는 방법도 있긴하겠지만, 그 도메인은 나중에 포트폴리오같은 페이지로 운영하면 좋지 않을까... 생각해본다. 왜냐하면, 만약에 나중에가서 도메인을 바꾸면 왠지 SEO 쪽에 문제가 생길 것 같기 때문이다.

디플로이는 surge 에 할 생각이다. 보통은 GitHub 페이지에서 deploy 하지만, 나중에 github 페이지로 사용되는 도메인은, 영어 블로그를 운영 할 때 사용 할 예정이다. 지금 당장은 미루고 있긴 한데, 언제 시작할지는 모르겠다.

블로그를 새로 시작하는게 조금 귀찮긴 하다 :) 이것 저것 설정 할 것도 많구..

블로그 테마는 나중에 Hexo 에 적응 다 끝내고, 시간나면 리뉴얼 하도록 해야겠다.
